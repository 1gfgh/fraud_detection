# Baseline

В данной папке находятся эксперименты с бустинговыми моделями для задачи обнаружения мошеннических транзакций. Основной используемой моделью является `CatBoostClassifier`.

## Содержимое папки

- `baseline.ipynb` — базовая версия модели CatBoost без сложного подбора гиперпараметров;
- `catboost_pro.ipynb` — улучшенная версия CatBoost с подбором гиперпараметров с помощью Optuna;

## Общая идея блока

В этом разделе проекта рассматривались бустинговые подходы как сильный табличный baseline для fraud detection.  
Модели обучались на транзакционном уровне, после чего предсказания агрегировались на уровень клиента.

Такая схема была выбрана потому, что итоговая задача интерпретировалась не только как поиск отдельных мошеннических транзакций, но и как выявление клиентов, у которых присутствует мошенническая активность. Для этого по каждому клиенту агрегировались:

- истинная метка — через максимум по транзакциям;
- предсказанный скор — через максимум по транзакциям.

Именно клиентский уровень использовался для финальной оценки качества.

## Baseline: `baseline.ipynb`

В ноутбуке `baseline.ipynb` была обучена базовая версия `CatBoostClassifier` с фиксированными гиперпараметрами и ранней остановкой по валидационной выборке.

### Основные особенности baseline-подхода

- использовался стандартный CatBoost без автоматического подбора параметров;
- в качестве eval-метрики при обучении использовался `AUC`;
- для получения финального бинарного решения на клиентском уровне применялся фиксированный порог **0.5**.

### Наиболее важные признаки

По итогам обучения наиболее значимыми признаками оказались временные характеристики транзакций:

1. `tx_day` — **21.76**
2. `tx_minute` — **13.52**
3. `tx_dayofweek` — **12.30**
4. `tx_hour` — **10.73**

Также заметный вклад вносили:

- `ProductId`
- `Amount`
- `ProviderId`
- `tx_month`

Это показывает, что в базовой модели существенную роль играли как календарно-временные паттерны, так и параметры самой транзакции.

### Результаты baseline на клиентском уровне

Тестовая выборка содержала:

- **749 клиентов**
- из них **9 мошенников**
- **740 честных клиентов**

- `TN = 737`
- `FP = 3`
- `FN = 3`
- `TP = 6`

Ключевые метрики на уровне клиента:

- **F0.5 = 0.6667**
- **PR-AUC = 0.7829**
- **ROC-AUC = 0.9659**
- **Precision = 0.6667**
- **Recall = 0.6667**
- **F1 = 0.6667**

Baseline-модель показала сильный стартовый результат и была использована как отправная точка для дальнейшего улучшения.

## Улучшенная версия: `catboost_pro.ipynb`

В ноутбуке `catboost_pro.ipynb` была реализована более продвинутая версия CatBoost с автоматическим подбором гиперпараметров с помощью **Optuna**.

### Что такое Optuna и зачем она использовалась

**Optuna** — это библиотека для автоматического подбора гиперпараметров модели.  
Вместо ручного перебора параметров она многократно обучает модель с разными комбинациями настроек и выбирает те, которые дают наилучший результат по заданной метрике.

В данном проекте Optuna использовалась для того, чтобы найти такую конфигурацию CatBoost, которая лучше всего оптимизирует именно **F0.5** на валидационной выборке.

Это особенно важно для данной задачи, потому что:

- основная метрика проекта — **F0.5**;
- `F0.5` сильнее штрафует за ложные срабатывания и делает больший акцент на **precision**;
- для fraud detection в ряде сценариев критично не просто находить мошенников, а минимизировать число ложных обвинений честных клиентов.

### Как проводился подбор

В `catboost_pro.ipynb`:

- запускалось **100 trial** в Optuna;
- использовался `TPESampler`;
- применялся `MedianPruner` для отсечения неудачных конфигураций;
- оптимизация шла по **F0.5** на валидации;
- дополнительно подбирался **оптимальный порог** классификации на клиентском уровне.

Подбирались параметры:

- `depth`
- `learning_rate`
- `l2_leaf_reg`
- `subsample`
- `iterations`
- `random_strength`
- `border_count`
- `bagging_temperature`
- `leaf_estimation_iterations`

### Лучшие найденные гиперпараметры

По итогам 100 запусков Optuna лучшая конфигурация дала:

- **Best F0.5 = 1.0000** на валидации

Лучшие параметры:

- `depth = 6`

- `bagging_temperature = 1.668084`

### Финальная оценка CatBoost Pro на тесте

## Основной вывод

Переход от базовой версии CatBoost к версии с подбором гиперпараметров через Optuna оказался успешным.

Несмотря на небольшое снижение `Recall`, модель `CatBoost Pro` значительно улучшила именно те характеристики, которые были наиболее важны в рамках проекта:

- выросла основная метрика **F0.5**;
- выросла **PR-AUC**;
- исчезли ложноположительные срабатывания (`FP = 0`);
- `Precision` достигла **1.0000**.

Таким образом, `catboost_pro.ipynb` можно считать основной и наиболее удачной бустинговой моделью в проекте, если приоритетом является высокая точность обнаружения мошенничества и минимизация ложных обвинений честных клиентов.
