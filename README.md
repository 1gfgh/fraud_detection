# Fraud Detection

## Описание проекта

Данный научно-исследовательский проект (НИР, 3 курс) посвящён задаче постфактум-детекции фродовых пользователей на основе истории транзакций.  
Цель работы — исследовать и сравнить несколько подходов к моделированию транзакционных данных разной степени структурированности и оценить их вклад в качество выявления мошеннических пользователей.  
В рамках проекта строятся модели, которые по истории транзакционной активности позволяют выявлять пользователей с повышенной вероятностью фродового поведения.

## Используемые данные

В качестве исходных данных используется датасет **Xente Fraud Detection Challenge**, доступный на платформе Zindi:

🔗 https://zindi.africa/competitions/xente-fraud-detection-challenge/data

Данный датасет содержит транзакционные данные сервиса Xente и предназначен для решения задачи бинарной классификации мошеннических операций.  
Он включает порядка 95 тысяч транзакций, совершённых в период с **15 ноября 2018 года по 15 марта 2019 года**. Подробности данных мы изучили в [data](data/README.md), а также провели [EDA](EDA/EDA.md).

## Архитектура решения

Проект состоит из двух логически независимых, но сравнимых между собой блоков. Каждый блок реализует собственный подход к обработке транзакционных данных и обучению модели. Исходно [рассматривается](baseline/README.md) базовый **CatBoost**, а также его дальнейшее улучшение за счёт подбора гиперпараметров.

### 1. Модель с полноценным структурированием транзакций

В рамках данного подхода поток транзакций преобразуется в структурированное представление графовыми методами.  
Основной акцент делается на выявлении связей между сущностями и моделировании транзакционной активности как графа взаимодействий, что позволяет учитывать сложные зависимости между объектами и восстанавливать структуру финансовых взаимодействий.

### 2. Последовательностные и табличные модели для анализа транзакционной истории

В данном блоке транзакционная история пользователя рассматривается как последовательность операций во времени, а также как набор агрегированных и табличных признаков.  
Подход включает как классические методы градиентного бустинга, так и deep learning архитектуры, ориентированные на выявление временных, поведенческих и скрытых паттернов фродовой активности.

## Распределение частей проекта

### Графовый блок
За данную часть отвечает **Долгих Даниил**.  
В этом направлении исследуются методы преобразования транзакционного потока в графовые структуры и построения моделей, учитывающих связи между участниками, операциями и другими сущностями.

### Последовательностный и tabular-блок
За данную часть отвечает **Бердов Игорь**.  
В этом направлении исследуются модели, работающие с транзакционной историей пользователя как с временным рядом или последовательностью событий. В рамках блока рассматриваются последовательностные deep learning подходы, включая **RNN**, **LSTM**, **GRU** и **transformer-based** архитектуры.
