# GRAPH

## Алгоритм построения графа

Построение графа основывается на 2 переменных. 

`Edge Keys`  и `node keys` - наборы параметров транзакций, на которых будет основываться группировка.

1. Группируем значения по `edge keys`, каждый набор будет представлять собой некий набор элементов "спуска" по дереву.
2. Каждую группу сортируем по времени транзакции.
3. Внутри каждой группы группируем элементы по `node keys`. Каждая подгруппа будет представлять из себя вершину нашего графа.

### Свойства

1. Такой граф всегда будет являться деревом, что позволяет применять большее количество подходов в анализе данных и обучении моделей.
2. Гибкая структура графа позволяет менять акцент в обучении модели без изменения общего пайплайна.
3. Граф отражает временную зависимость транзакций и сразу несет в себе элемент обучения на последовательностях, что необходимо в контексте анализа транзакций.
